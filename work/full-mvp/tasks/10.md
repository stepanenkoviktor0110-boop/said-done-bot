# Task 10: Unit tests — db_ops, task_extractor, debounce, survey

## Description
Write unit tests for core modules.

## What to test

### db_ops
- upsertUser: creates with trial_remaining=20, updates existing
- createVoiceRequest: inserts, returns row
- updateVoiceRequest: updates action_type, summary_length, task_count
- decrementTrial: 20→19, doesn't go below 0
- completeSurvey: phase=2, trial_remaining=10
- block_survey: survey_blocked=1

### task_extractor
- _parse_response: numbered list → tasks array
- _parse_response: __NO_TASKS__ → marker
- _parse_response: __TOO_MANY_TASKS__ → marker
- extract_tasks: truncation at 4000 chars (mock LLM)

### debounce buffer
- Single voice: immediate processing (after 3s timer)
- 3 voices within 3s: combined transcript, one result
- Voice after 3s timer: new buffer, separate processing
- Batch limit: 12 voices → only 10 processed

### survey
- Heuristic gate: Q1-Q3 reject <5 words, Q4 accept "4"
- 2-strike rule: 2 garbage answers → blocked
- Fail-open: LLM error → answer accepted

## Verification

```bash
pytest tests/ -v
```
