# Task 9: Survey flow — 4 questions, sanity check, 2-strike rule, fail-open

## Description
Implement trial survey: sequential questions with heuristic + LLM adequacy checks.

## What to do

In `entrypoints/telegram_bot.py`:
1. `async enter_survey(ctx, user)` — send SURVEY_INTRO + question 1, set in_survey=True
2. Text handler for survey answers (check `in_survey` flag):
   - Get question index from `user["survey_progress"]`
   - Apply heuristic gate: Q1-Q3 ≥5 words, not punctuation-only; Q4 accept digit 1-5
   - If heuristic fails: LLM sanity check (fail-open if LLM unavailable)
   - If inadequate: increment retry; if retry≥1 → block_survey, SURVEY_BLOCKED
   - If adequate: save_survey_response, advance_survey, ask next question
   - If question 4 answered adequately: complete_survey, SURVEY_COMPLETE

3. LLM sanity check prompt:
   "Ты проверяешь ответы на опрос о Telegram-боте. Определи: ответ осмысленный и по теме, или это мусор. Ответь одним словом: 'адекватный' или 'неадекватный'."

## Verification
Manual: exhaust trial → survey → 4 adequate answers → +10 credits. DB: survey_responses rows, trial_phase=2.
